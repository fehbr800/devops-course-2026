# Introduction to DevOps Culture

Instructor: Breno Felix de Sousa  
Course: Information Systems  

This repository is used to manage the activities and projects of the course **Introduction to DevOps Culture**.

The course is organized as a **project-based learning experience**, where students will explore DevOps practices through a **literature review and analysis of DevOps tools and practices used in industry**.

---

# Course Objective

The goal of this course is to introduce students to the **DevOps culture, practices, tools, and workflows** used in modern software engineering.

Throughout the semester, students will:

- investigate DevOps practices in the literature
- analyze tools used in DevOps pipelines
- design a conceptual DevOps pipeline
- produce a final report summarizing their findings

---

# Course Methodology

The course is structured as a **progressive project**, where each week represents a step in the development of a **DevOps literature review and pipeline design**.

Students will work in **groups** and submit weekly progress.

All activities are managed using **GitHub**, simulating a real-world DevOps workflow.

---

# Weekly Activities

The project is divided into the following stages:

### Week 01 – Topic definition
Define the DevOps topic your group will investigate.

### Week 02 – Research question
Define a clear research question for your study.

### Week 03 – Article search
Search and collect relevant academic papers related to your topic.

### Week 04 – Article selection
Select the most relevant articles for analysis.

### Week 05 – Article summary
Summarize the selected papers.

### Week 06 – Evidence table
Organize findings in an evidence table.

### Week 07 – DevOps practices
Identify DevOps practices mentioned in the literature.

### Week 08 – DevOps tools
Identify tools used in DevOps pipelines.

### Week 09 – DevOps pipeline
Design a conceptual DevOps pipeline.

### Week 10 – Draft report
Prepare the first version of the report.

### Week 11 – Final report
Submit the final report.

# Submission Workflow

The submission process follows a DevOps-style workflow:

Create branch

↓

Add submission file

↓

Commit changes

↓

Open Pull Request

↓

Automatic validation (GitHub Actions)

↓

Instructor review

↓

Merge


All Pull Requests must pass the **automatic validation pipeline** before being accepted.

---

# Templates

To help students structure their work, the repository provides templates in the folder: templates/


Available templates include:

- Evidence table

- DevOps pipeline template

Students are encouraged to use these templates when preparing their submissions.

---

# Continuous Integration

This repository uses **GitHub Actions** to automatically validate submissions.

Before a submission can be merged:

- the file structure must be correct
- the pipeline validation must pass

This simulates real-world DevOps practices.
